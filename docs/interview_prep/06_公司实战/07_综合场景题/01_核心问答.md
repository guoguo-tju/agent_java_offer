算法题1: 
实现一个上台阶的游戏
一次可能走一步，可能走两步
走n级台阶的所有可到达的具体执行步骤,这个是 LeetCode 的哪个题？题目是什么？

可以，我们就按“输出所有具体步骤”这个版本来讲。

思路先说人话：

1. 你每次只能走 `1` 或 `2`。
2. 所有路径本质上是：找所有由 `1/2` 组成、且总和等于 `n` 的序列。
3. 这就是典型回溯：

- 维护当前路径 `path`  
- 维护剩余台阶 `remain`  
- `remain == 0`：找到一条完整方案，加入结果  
- `remain < 0`：这条路走过头了，返回  
- 否则尝试走 `1` 和 `2`，递归后撤销（回溯）

Java 代码如下：

```java
import java.util.*;

public class StairPaths {
    public List<List<Integer>> allWays(int n) {
        List<List<Integer>> res = new ArrayList<>();
        List<Integer> path = new ArrayList<>();
        dfs(n, path, res);
        return res;
    }

    private void dfs(int remain, List<Integer> path, List<List<Integer>> res) {
        if (remain == 0) {
            res.add(new ArrayList<>(path)); // 注意拷贝
            return;
        }
        if (remain < 0) return;

        // 走1步
        path.add(1);
        dfs(remain - 1, path, res);
        path.remove(path.size() - 1);

        // 走2步
        path.add(2);
        dfs(remain - 2, path, res);
        path.remove(path.size() - 1);
    }

    public static void main(String[] args) {
        StairPaths s = new StairPaths();
        System.out.println(s.allWays(4));
        // 输出: [[1,1,1,1],[1,1,2],[1,2,1],[2,1,1],[2,2]]
    }
}
```

补一句面试点：  
这个题如果只要“方案数”，可用 DP（LeetCode 70）；如果要“具体步骤”，就用回溯。

如果只要“方案数”，思路比回溯简单很多：用 DP。

思路
设 dp[i] = 到第 i 阶的方案数。
到第 i 阶只有两种来源：

从 i-1 走 1 步上来
从 i-2 走 2 步上来

```java
class Solution {
    public int climbStairs(int n) {
        if (n <= 1) return 1;

        int[] dp = new int[n + 1];
        dp[0] = 1; // 到第0阶：1种（不动）
        dp[1] = 1; // 到第1阶：1种（走1步）

        for (int i = 2; i <= n; i++) {
            dp[i] = dp[i - 1] + dp[i - 2];
        }

        return dp[n];
    }
}
```

---

### 这是最常见 DP 模板思路

1. **定义状态**：`dp[i]` 表示到第 `i` 阶的方法数。
2. **初始化**：`dp[0]=1, dp[1]=1`。
3. **状态转移**：`dp[i]=dp[i-1]+dp[i-2]`。
4. **返回答案**：`dp[n]`。

---

### 为什么 `dp[0]=1`？

这是一种“空方案”的定义，方便统一转移公式。  
比如算 `dp[2]` 时：`dp[2]=dp[1]+dp[0]=1+1=2`，刚好得到 `[1,1]` 和 `[2]`。

算法题2: 
已知F3（），随机生成0，1,2的其中一个, 概率均匀的
构造F5（），随机生成0，1, 2, 3, 4中的一个，要求概率也是均匀的,

有的，这题最稳的思路就是 **拒绝采样**，我们一起过一下：

1. `F3()` 均匀给 `0/1/2`。
2. 调两次 `F3()`，构造一个 `0~8` 的均匀数：
  `x = a * 3 + b`（`a,b` 都来自 `F3()`）。  
   这样 9 种结果等概率。
3. 如果 `x < 5`，直接返回 `x`（就是 `0~4`）。
4. 如果 `x >= 5`，丢弃重来。

为什么对：  

- `0~8` 每个值概率都是 `1/9`。  
- 条件在 `x<5` 下，`0~4` 仍等概率。

期望调用次数：  

- 接受概率 `5/9`，平均循环 `9/5=1.8` 次。  
- 每轮用 2 次 `F3()`，所以平均约 `3.6` 次 `F3()`。

简版伪码：

```java
int f5() {
    while (true) {
        int a = f3(); // 0..2
        int b = f3(); // 0..2
        int x = a * 3 + b; // 0..8
        if (x < 5) return x;
    }
}
```

12. 你怎么用 AI 提升研发效率
我会从“研发全链路”来用 AI，而不是只让它写代码：
1. 需求阶段：让 AI 把 PRD 转成技术方案草稿、接口清单、边界条件和风险列表，先把歧义消掉。  
2. 开发阶段：用 AI 生成样板代码、SQL、脚手架和重构建议，人主要做架构决策和关键逻辑。  
3. 测试阶段：自动补单测/集成测试用例，特别是异常分支、边界值、回归用例。  
4. 评审阶段：让 AI 先做第一轮 Code Review（空指针、并发、事务、幂等、性能热点），再人工做最终把关。  
5. 沉淀阶段：自动生成技术文档、变更记录和复盘报告，减少“做了但没沉淀”。

“我把 AI 当研发副驾驶，负责提速和补盲；人负责架构决策和质量门禁，用流程和指标保证提效不降质。”

1. **接入层**  
API Gateway + 鉴权（JWT）+ 限流（用户/IP/设备）+ 灰度路由。  
长响应用 SSE/WebSocket，短请求走 HTTP。

2. **编排层（Agent Runtime）**  
把流程做成状态机/图：意图识别 -> 检索/工具 -> 生成 -> 审核 -> 返回。  
每个节点有超时、重试、最大步数、熔断和降级。

3. **模型层**  
模型路由（大模型/小模型切换）。  

4. **能力层**  
Prompt 模板中心、Tool 调用、RAG 检索服务（向量+关键词混检）。  

5. **数据层**  
MySQL（用户/订单/任务状态）、Redis（会话和缓存）、对象存储（文件/产物）、向量库（知识检索）。  
关键写操作必须幂等键，避免重试导致重复副作用。

6. **安全与治理层**  
输入输出内容安全（涉政/涉黄/隐私）、提示词注入防护、工具白名单、最小权限、审计日志全链路可追溯。

Metrics + Logs + Traces 三件套；重点盯成功率、时延、成本、幻觉率、工具失败率。
