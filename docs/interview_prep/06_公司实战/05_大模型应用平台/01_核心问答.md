## 1. 大模型应用平台

### 资料说明

- 本节内容来自个人面试资料的重组整理与公开资料归纳。
### 原文内容

我全网看下来，**“某大模型应用平台产品线的 Agent 开发一面”公开直接同名资料不算多**，但把 **字节 AI Agent 开发一面、某大模型应用平台大模型一面、某大模型应用平台后端/AI 后端一面、以及某大模型应用平台 Agent 相关 JD** 拼起来，画像已经比较清楚了：**一面最常见的是“项目深挖 + Agent/RAG 细节 + 一点模型基础 + 一点工程基础/手撕”**，而且面试官很爱连续追问，不太接受泛泛而谈。([牛客网](https://www.nowcoder.com/feed/main/detail/36f397670374455896bd05de6858c06a))

**岗位侧重点**也比较明确。公开职位描述里，某大模型应用平台 Agent 技术研发会做 **Multi-Agent 框架、Memory 机制**，还会做 **UI-Agent、DeepResearch Agent**；另外字节的 Agent 相关岗位也明显要求你熟悉 **Agentic AI、RAG、Workflow、多 Agent 协作、模型效果与性能评估**。这意味着你今天下午这一面，大概率不会只问“会不会调 API”，而是会看你能不能把 Agent 做成完整系统。([BOSS直聘](https://m.zhipin.com/job_detail/41ac5d50f49e184503B_3d-1EVJU.html?utm_source=chatgpt.com))

我建议你现在优先刷这 12 个问题，命中率最高：  
1）你做的 Agent 为什么是这个架构，不是别的；  
2）单 Agent 和多 Agent 怎么选，子 Agent 如何分工；  
3）tool 调错时怎么兜底；  
4）memory 怎么分层设计；  
5）skills / MCP / RAG 各自解决什么问题；  
6）上下文过长怎么处理；  
7）知识库从入库到回答的完整链路；  
8）多路召回怎么做权重和评估；  
9）你们为什么选这个框架，比如 LangChain / LangGraph / 自研；  
10）如何评测 Agent 效果；  
11）你项目里最难的一个工程问题是什么；  
12）一道常规算法题和一两个 Java/后端基础问题。这个清单是我根据多份字节/某大模型应用平台/Agent 一面真实问题汇总出来的。([牛客网](https://www.nowcoder.com/feed/main/detail/36f397670374455896bd05de6858c06a))

如果你想要一个更直接的判断：**这一面最像“拿你的项目当主线，顺手检查你是不是真的懂 Agent 系统，而不是只会搭 demo”**。尤其某大模型应用平台 Agent 的 JD 本身就对 **Multi-Agent、Memory、DeepResearch/UI-Agent、评估与工程落地** 有明确要求，所以你的回答一定要从“系统怎么设计、为什么这么设计、怎么评估、怎么做稳定性和安全性”这几个维度展开。([BOSS直聘](https://m.zhipin.com/job_detail/41ac5d50f49e184503B_3d-1EVJU.html?utm_source=chatgpt.com))

你现在最值钱的准备顺序是：**先把你最强项目用 3 分钟、8 分钟、15 分钟三种版本讲顺，再把 RAG、memory、tool calling、multi-agent、skills、评测体系 各准备一个标准答案，最后过一遍 Java/Redis/MySQL/算法手撕。** 这样最贴近我搜到的真实一面分布。([牛客网](https://www.nowcoder.com/feed/main/detail/36f397670374455896bd05de6858c06a))

你要的话，我下一条直接给你做一份 **“某大模型应用平台 Agent 开发一面冲刺题 30 题 + 参考回答提纲”**。
