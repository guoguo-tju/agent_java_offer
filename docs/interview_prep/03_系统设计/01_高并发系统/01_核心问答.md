- 当日产出：
  - 口述“令牌桶 vs 漏桶 vs 滑动窗口”
  - 追问准备：“分布式限流怎么做一致性？”
- 答案:

**Sentinel（阿里开源）** 这条体系。  

它的本质是：**控制面下发规则，数据面本地快速判定**。

**大致原理**

1. **控制面**：规则配置与治理

- 在控制台配置限流/熔断/系统保护规则。  
- 规则存到配置中心（如 Nacos），动态推送到各实例。

1. **数据面**：请求经过本地拦截链

- 业务代码在资源点埋点（接口、方法、下游调用）。  
- 每次请求进来走 Sentinel 的 slot chain（统计、限流、熔断等）。  
- 本地内存计数 + 滑动时间窗，判断当前 QPS/RT/异常率是否超阈值。  
- 超阈值就执行策略：快速拒绝、排队等待、预热放量等。

1. **为什么快**

- 判定在本机完成，不走远程 RPC；  
- 规则热更新，生效快；  
- 指标异步上报控制台做可视化和告警。

面试可一句话总结：  

“Sentinel 类方案是控制面统一治理、数据面本地判定的架构，通过滑动窗口统计+规则引擎在毫秒级完成限流/熔断决策，兼顾性能和可运维性。”

- 分布式限流怎么做一致性？

集群流控模式（Cluster Flow）Sentinel 用 Token Server + Token Client。

- Client 收到请求先向 Token Server 申请令牌。
- Server 按规则（QPS、匀速排队、预热等）统一判定是否放行。
- 这样能把多个实例的流量收敛到一个“全局配额视角”，一致性明显比本地模式好。

“Sentinel 默认是单机限流；要做分布式一致限流要开集群流控，用 token server 统一发令牌。它在性能和一致性之间做工程折中，适合互联网场景。”

### 原文内容

### 如何设计一个高并发系统

常见回答框架可以按“拆、缓存、削峰、分治、兜底”来讲：

1. **系统拆分**：把大系统拆成多个子系统，单机单库变成分布式架构，让每个子系统只承接一部分压力。
2. **缓存**：高并发大多是读多写少，优先把热点数据放缓存，让大部分请求不落数据库。
3. **MQ 削峰**：大量写请求先灌入 MQ 排队，后端按自己承载能力慢慢消费，避免把数据库直接打死。
4. **分库分表**：当单库单表已经成为瓶颈时，通过水平拆分摊平写入和存储压力。
5. **读写分离**：主库负责写，从库负责读，读流量大时继续扩从库。
6. **ES / 检索引擎承接查询**：全文检索、复杂搜索、部分统计类查询迁到 ES，减轻主库压力。
7. **工程兜底**：限流、防刷、CDN、静态化、异步化、小事务、锁粒度优化、JVM/SQL 优化、灰度扩容。

如果是秒杀/大促场景，还要强调：
- 秒杀前不要暴露真实秒杀地址；
- 前端尽量静态化；
- 网关和 Redis 做防刷限流；
- 库存预热到 Redis，用 Lua 做原子扣减；
- 能异步的都异步，比如短信、积分、优惠券等；
- 事务尽量小，不要把写日志、调接口这些操作包进大事务；
- 乐观锁、悲观锁、间隙锁、快照读、当前读、MVCC 这些并发控制细节必须清楚，否则数据库高峰期很容易被锁竞争拖垮。

## 补充原文：分布式请求顺序性如何保证

### 原文内容

### 分布式服务接口请求的顺序性如何保证？

首先，工程上更推荐的思路是：**尽量从业务设计上避免强顺序依赖**。因为一旦引入严格顺序保证，通常会带来：
- 系统复杂度上升；
- 吞吐下降；
- 热点 key 压力变大；
- 排障和恢复都更麻烦。

如果业务确实要求顺序性，常见解法有三类：

1. **Hash 分发到同一机器**
- 比如按订单 ID 做一致性 hash，把同一业务对象的请求路由到同一台机器；
- 到了该机器内部，再把同一 key 的请求扔到内存队列里串行处理；
- 这样能做到“同一 key 有序，不同 key 并行”。

2. **MQ / 内存队列串行化**
- 生产端按业务 key 发到同一个 partition 或同一条队列；
- 消费端再对 key 做 hash，落到固定线程或固定内存队列里顺序消费。

3. **锁与序列号控制**
- 可以通过分布式锁、数据库行锁，或者请求携带版本号 / 序列号来控制乱序更新；
- 但这种做法更重，吞吐和热点问题会更明显。

如果是更新型请求，还可以在各阶段加幂等保护，比如：
- 执行前先检查 redis 幂等 key；
- 执行过程中对异常做回滚和恢复；
- 成功后再修改状态位，避免重复执行。

面试里可以总结成一句：
**顺序性不是白拿的，核心思路是“同一 key 聚合到同一路径上串行处理”，常见手段是 hash 路由 + 队列串行 + 幂等兜底。**
