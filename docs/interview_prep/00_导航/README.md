## 这页解决什么问题

它主要解决 3 件事：

1. **不知道先看哪块**
2. **不知道不同目录分别放什么**
3. **想按目标定制复习路径**

## 推荐使用顺序

1. 先看 [总索引](总索引.md)
   看全局目录和每一块分别讲什么
2. 再看 [复习路线图](复习路线图.md)
   按你的目标选复习顺序

## 如果你现在的目标很明确

### 目标 1：Java 后端转 AI Agent

建议顺序：

1. `01_AI`
2. `02_后端`
3. `03_系统设计`
4. `05_项目表达`

重点不是只背概念，而是把：
- Agent / RAG / 评测 / 安全
- 后端基础设施能力
- 项目表达

串成一条能讲通的主线。

### 目标 2：后端面试冲刺

建议顺序：

1. `02_后端`
2. `03_系统设计`
3. `04_算法`
4. `05_项目表达`

重点是把后端八股和系统设计拉通，不要只会背知识点。

### 目标 3：项目口述和场景题强化

建议顺序：

1. `05_项目表达`
2. 发现不会展开的点，再反查 `01_AI / 02_后端 / 03_系统设计`

这种路径最适合“临近面试，需要把项目说顺”的场景。

## 这套目录的组织原则

这套资料统一按：

**方向 -> 主题 -> 单文件问答**

组织。

具体含义是：

1. 先按大方向分层
   例如 AI、后端、系统设计、算法、项目表达
2. 每个方向下再按主题拆分
   例如 RAG、Redis、Kafka、交易风控、邀请码系统
3. 每个主题主要只维护一个主文件
   即 `01_核心问答.md`

这样做的目的，是让你在复习时减少来回跳文件的成本。
